一、产品概述

1.1 产品定位

教师作业批改助手是一款基于 DeepSeek 大模型的智能批改 AI Agent，面向中小学教师提供多学科作业批改服务。产品将提示词工程的最佳实践（评分规则、评分锚点、错误约束、统一输出包装器）封装为简单易用的 Web 界面，教师无需任何技术背景即可使用。

1.2 核心价值

痛点解决方案
批改作业耗时巨大，语文作文每篇需10-15分钟AI 30秒完成批改，教师只需审核确认
评分标准因人而异，缺乏一致性内置标准化评分规则+评分锚点，全校统一标准
评语泛泛而谈，"继续努力"式套话评语强制引用学生原文+给出可操作步骤
数学批改只对答案不看过程逐步骤验证逻辑链，定位具体错误步骤和原因
英语批改只划红线不解释每个错误附中文解释+地道修正+学习建议
全班学情靠经验判断，缺乏数据支撑自动生成含统计表格+分层建议+行动计划的数据报告

2.3 API 安全性

安全说明：API Key 仅存储在用户浏览器的 localStorage 中，不会上传到任何服务器。所有 API 调用直接从用户浏览器发起，不经过任何中转。这意味着你的密钥始终在你自己的设备上，不存在被第三方窃取的风险。每次批改仅传输当前作业内容到 DeepSeek 服务器，不保留任何历史数据。

4.2 各学科操作详解

📝 语文读后感批改

输入：学生《背影》读后感全文（推荐300-800字） 可选：作文标题、学生姓名 输出：总分(0-100) + 四维度分项得分 + 扣分详情 + 评语(总体评价/亮点/待改进/改进建议) 耗时：约15-30秒

📐 数学解答题批改

必填：① 题目完整描述 ② 标准答案/关键证明步骤 ③ 学生解答过程 可选：学生姓名 输出：正误判定 + 步骤统计 + 得分率 + 错误步骤定位 + 错误类型 + 错误原因详解 + 完整正确解法 耗时：约20-35秒

🔤 英语作文批改

输入：学生英语作文全文（推荐80-300词） 可选：作文标题、学生姓名 输出：整体水平等级 + 四类错误统计 + 每个错误的原文/中文解释/修正 + 修改后全文 + 学习建议 耗时：约20-30秒

📊 学情分析报告

输入：全班多份批改结果的JSON数组（需先用语文模板逐份批改后收集） 输出：Markdown格式的七模块分析报告（统计/诊断/优劣势/分层建议/行动计划） 耗时：约30-45秒

六、内置提示词模板说明

Agent 内置四套提示词模板，均遵循 RCT（角色-上下文-任务）框架，输出统一包装器 JSON。每套模板的评分规则、输出格式、错误约束均为固定设计，仅 {变量} 部分每次替换。

6.1 模板参数表

模板变量必填说明
语文{作业内容} {作文标题} {学生姓名}作业内容必填学生读后感全文
数学{作业内容} {题目} {标准答案} {学生姓名}前三项必填题目+标准答案+学生解答
英语{作业内容} {作文标题} {学生姓名}作业内容必填学生英语作文全文
学情分析{批改结果JSON数组}必填全班批改结果数组

6.2 统一输出包装器

{ "success": true/false, // 批改是否成功 "error_message": null/"…", // 错误描述（success=false时） "data": { … }, // 学科特定批改结果 "metadata": { // 元信息 "subject": "chinese|math|english|analysis", "timestamp": "2026-06-08T…", "version": "3.0" } }

6.3 错误处理机制

错误场景Agent 响应
API Key 未填写弹出提示："请先在左侧填写 DeepSeek API Key"
API Key 无效对话区显示："❌ 批改出错 — 请检查 API Key 是否正确"
输入内容为空弹出提示："请输入作业内容"
学科不匹配模型返回 success=false，对话区显示错误原因（如"检测到输入为英语作文，当前为数学批改器"）
JSON 解析失败对话区显示："❌ 批改出错 — 请检查输入内容格式"

七、常见问题（FAQ）

问题解答
如何获取 DeepSeek API Key？访问 platform.deepseek.com → 注册/登录 → 点击左侧"API Keys" → "创建 API Key" → 复制密钥。新用户通常有免费额度。
每次批改费用多少？DeepSeek 定价约 ¥1/百万 tokens。单次批改消耗约2000-4000 tokens（含提示词+输出），约¥0.002-0.004/次。批改100篇作文约¥0.2-0.4。
API Key 是否安全？密钥仅存储在你的浏览器 localStorage 中，不会上传到任何服务器。所有 API 调用直接从你的浏览器发起。如果你部署到 GitHub Pages，请注意仓库必须是 Public 的（免费方案），但 API Key 不在代码中——每个用户在自己的浏览器中输入自己的密钥。
能否用于其他课文（非《背影》）？语文模板当前专为《背影》读后感优化（评分规则中包含"中国式父爱""爬月台"等特定引用）。如需用于其他课文，需修改评分规则中与原文具体内容相关的描述。数学和英语模板无此限制，可通用。
支持批量批改吗？当前版本为单次批改模式。如需批量批改，建议使用 API 脚本循环调用（见 §8 开发者接口），或先逐份批改收集结果，再用"学情分析报告"功能进行宏观分析。
手机能用吗？完全支持。界面采用响应式设计，在手机浏览器中自动适配为单列布局。侧边栏折叠到顶部，对话区和输入区占满屏幕。
批改不准确怎么办？这是辅助工具，最终评判权在教师手中。如果发现系统性偏差（如总是对某类错误评分偏松），可以调整提示词模板中的评分锚点。模板代码在 index.html 的 PROMPTS 对象中，可直接编辑。
能否添加更多学科？可以。复制现有模板结构，按照 RCT 框架编写新学科的提示词，然后在 UI 中添加对应的学科按钮和渲染函数即可。

八、开发者接口（API 脚本调用）

如需在代码中批量调用批改服务，可直接使用以下 Python 示例：

import requests, json API_KEY = "sk-your-key-here" API_URL = "https://api.deepseek.com/chat/completions" def grade_essay(essay_text, student_name=""): """语文读后感批改""" prompt = f"""# 角色…(完整提示词模板)…学生作文：{essay_text}""" resp = requests.post(API_URL, headers={"Authorization": f"Bearer {API_KEY}"}, json={"model":"deepseek-chat","messages":[{"role":"user","content":prompt}], "temperature":0.3,"max_tokens":4096}) raw = resp.json()["choices"][0]["message"]["content"] return json.loads(raw) # 返回统一包装器JSON # 使用 result = grade_essay("读完《背影》后，我挺感动的…", "张三") print(f"总分：{result['data']['总分']}") print(f"评语：{result['data']['评语']['总体评价']}")
